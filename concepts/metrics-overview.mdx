## What Are Metrics?

LangCheck metrics are functions that evaluate the quality of text generated by LLMs. Each metric function takes text as input and returns a `MetricValue` object containing scores, explanations, and metadata.

Metrics work with any LLM and any library - they simply evaluate lists of strings:

```python
import langcheck

# Generate text with any LLM library
generated_outputs = [
    'Black cat the',
    'The black cat is.',
    'The black cat is sitting',
    'The big black cat is sitting on the fence'
]

# Evaluate text quality
fluency_scores = langcheck.metrics.fluency(generated_outputs)
```

## The MetricValue Object

All LangCheck metric functions return a `MetricValue` object - a rich data structure that goes beyond simple numeric scores. It contains:

- **metric_name**: The name of the metric (e.g., "fluency", "toxicity")
- **metric_values**: A list of numeric scores for each input
- **metric_inputs**: The original inputs (prompts, generated outputs, references, etc.)
- **explanations**: Optional explanations for each score
- **language**: The language of the evaluated text
- **token_usage**: Token usage information when using remote LLMs

### Converting to DataFrame

Use the `to_df()` method to get a pandas DataFrame for analysis:

```python
fluency_scores = langcheck.metrics.fluency(generated_outputs)
df = fluency_scores.to_df()
print(df)
```

The DataFrame includes all inputs and outputs, making it easy to analyze results:

| generated_outputs | explanations | metric_values |
|------------------|--------------|---------------|
| Black cat the | None | 0.23 |
| The black cat is. | None | 0.67 |
| The black cat is sitting | None | 0.89 |
| The big black cat is sitting on the fence | None | 0.95 |

### Visualizing Results

The `MetricValue` object provides built-in visualization methods:

#### Scatter Plot

Create interactive scatter plots to explore your data:

```python
fluency_scores.scatter()
```

This shows each data point with its metric score, allowing you to hover over points to see the full text.

#### Histogram

Visualize the distribution of scores:

```python
fluency_scores.histogram()
```

<Note>
Visualization methods work best in Jupyter notebooks, VS Code notebooks, or Google Colab.
</Note>

### Comparing Multiple Metrics

You can visualize relationships between different metrics:

```python
import langcheck

toxicity_scores = langcheck.metrics.toxicity(production_outputs)
fluency_scores = langcheck.metrics.fluency(production_outputs)

# Compare two metrics
langcheck.plot.scatter(toxicity_scores, fluency_scores)
```

## Threshold Operations

One of the most powerful features of `MetricValue` is threshold operations. You can use comparison operators to create pass/fail criteria:

```python
fluency_scores = langcheck.metrics.fluency(generated_outputs)

# Apply threshold
results = fluency_scores > 0.5
print(results)
```

This returns a `MetricValueWithThreshold` object that shows:

- **Pass Rate**: The percentage of outputs that meet the threshold
- **threshold_results**: Boolean values indicating which outputs pass
- All original `MetricValue` data

### Supported Comparison Operators

All standard comparison operators are supported:

```python
fluency_scores > 0.5   # Greater than
fluency_scores >= 0.5  # Greater than or equal
fluency_scores < 0.5   # Less than
fluency_scores <= 0.5  # Less than or equal
fluency_scores == 1.0  # Equal to
fluency_scores != 0.0  # Not equal to
```

### Pass Rate

The `pass_rate` property shows what percentage of outputs meet the threshold:

```python
results = fluency_scores > 0.5
print(f"Pass rate: {results.pass_rate * 100}%")
# Output: Pass rate: 75.0%
```

### Boolean Methods

Check if all or any outputs meet a condition:

```python
results = fluency_scores > 0.5

# Check if all outputs pass
if results.all():
    print("All outputs are fluent!")

# Check if any outputs pass
if results.any():
    print("At least one output is fluent")
```

These methods are especially useful for binary metrics like text structure validation:

```python
json_check = langcheck.metrics.is_json_object(generated_outputs)

if json_check.all():
    print("All outputs are valid JSON")
```

## Using Metrics with Assertions

Metrics integrate seamlessly with Python's `assert` statement, making them perfect for unit tests:

```python
import langcheck

def test_fluency():
    outputs = generate_text_from_llm()
    # Assert that all outputs have fluency > 0.5
    assert langcheck.metrics.fluency(outputs) > 0.5

def test_no_toxicity():
    outputs = generate_text_from_llm()
    # Assert that all outputs have toxicity < 0.1
    assert langcheck.metrics.toxicity(outputs) < 0.1

def test_json_format():
    outputs = generate_json_from_llm()
    # Assert that all outputs are valid JSON objects
    assert langcheck.metrics.is_json_object(outputs).all()
```

When an assertion fails, you get clear error messages showing which outputs didn't meet the threshold.

### Test Example with References

For reference-based metrics, you can compare against ground truth:

```python
import langcheck
from langcheck.utils import load_json

def test_semantic_similarity():
    prompts = load_json('test_prompts.json')
    reference_outputs = load_json('reference_outputs.json')
    generated_outputs = [my_llm_app(p) for p in prompts]
    
    # Ensure generated outputs are semantically similar to references
    similarity = langcheck.metrics.semantic_similarity(
        generated_outputs, 
        reference_outputs
    )
    assert similarity > 0.9
```

## Single String Input

Metric functions also accept single strings, which is useful for guardrails and real-time evaluation:

```python
# Evaluate a single output
user_input = "What's the weather today?"
result = langcheck.metrics.toxicity(user_input)

if result.metric_values[0] > 0.5:
    print("Warning: Toxic input detected")
```

## Accessing Raw Values

You can access the raw metric values directly:

```python
fluency_scores = langcheck.metrics.fluency(generated_outputs)

# Get the list of scores
scores = fluency_scores.metric_values
print(scores)  # [0.23, 0.67, 0.89, 0.95]

# Access individual scores
first_score = fluency_scores.metric_values[0]
print(f"First output score: {first_score}")
```

## Handling None Values

Some metrics may return `None` values when a score cannot be computed (e.g., text too long for local models, API errors):

```python
scores = langcheck.metrics.sentiment(very_long_outputs)
# metric_values might be: [0.8, None, 0.6, 0.9]

# Threshold operations treat None as failing
results = scores > 0.5
# None values will have threshold_result = False
```

<Warning>
When using threshold operations, `None` values in `metric_values` will always result in `False` for threshold results.
</Warning>
